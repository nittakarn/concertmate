## Section 1 — Persona & Need

### Persona
**Concert Goers (มีตั๋วแล้ว)**

**Context**
ผู้ที่ซื้อตั๋วคอนเสิร์ตแล้วและกำลังวางแผนเตรียมตัวก่อนวันงาน มีงบประมาณ รูปแบบการเดินทาง และความต้องการที่แตกต่างกัน ต้องการข้อมูลที่ครบถ้วน เชื่อถือได้ และสามารถวางแผนทุกอย่างได้ในที่เดียว เพื่อลดเวลาในการค้นหา ลดความกังวล และเตรียมตัวได้อย่างมั่นใจก่อนวันงาน

**Goal**
สามารถวางแผนทริปคอนเสิร์ตได้ครบในที่เดียว ตั้งแต่งบประมาณ ที่พัก การเดินทาง การเตรียมตัว และกฎของงาน เพื่อประหยัดเวลา ลดความกังวล และควบคุมค่าใช้จ่ายให้อยู่ภายในงบประมาณที่กำหนด

**Behaviours**
- กำหนดงบประมาณสำหรับทั้งทริป
- ค้นหาโรงแรมใกล้สถานที่จัดงาน
- เปรียบเทียบเส้นทางและขนส่งสาธารณะ
- อ่านรีวิวจาก Facebook, X, TikTok และ Pantip
- ศึกษากฎและข้อกำหนดของงาน
- เตรียมสิ่งของที่จำเป็นก่อนวันงาน
- ตรวจสอบสภาพอากาศล่วงหน้า

**Frustrations**
- ข้อมูลกระจัดกระจายอยู่หลายแพลตฟอร์ม
- ต้องคำนวณค่าใช้จ่ายทั้งหมดด้วยตนเอง
- ไม่มั่นใจว่าข้อมูลที่พบเป็นข้อมูลล่าสุดและเชื่อถือได้
- กังวลว่าจะลืมของสำคัญหรือทำผิดกฎของสถานที่จัดงาน
- ไม่แน่ใจว่าควรออกเดินทางเมื่อไร และจะเดินทางกลับอย่างไรหลังงานเลิก

---

### Need

**Pain**
- ข้อมูลที่จำเป็นสำหรับการวางแผนอยู่หลายแหล่ง ทำให้เสียเวลาค้นหาและเปรียบเทียบ
- ไม่ทราบค่าใช้จ่ายรวมของทั้งทริป ส่งผลให้ควบคุมงบประมาณได้ยาก
- ไม่รู้ว่าควรเลือกโรงแรมหรือเส้นทางเดินทางแบบใดจึงจะเหมาะสม
- กังวลว่าจะพลาดข้อมูลสำคัญ เช่น ของต้องห้าม กฎของงาน หรือการเดินทางกลับ

**Gain**
- ได้ข้อมูลสำคัญทั้งหมดในที่เดียว
- เห็นภาพรวมค่าใช้จ่ายก่อนเดินทาง
- ลดเวลาในการค้นหาและวางแผน
- เตรียมตัวได้ครบถ้วนก่อนวันงาน
- ตัดสินใจและเตรียมตัวก่อนวันงานได้อย่างมั่นใจ พร้อมควบคุมค่าใช้จ่ายให้อยู่ในงบประมาณ

**Business Outcome**
- เพิ่มอัตราการกลับมาใช้งาน (Repeat Usage)
- เพิ่ม Organic Word-of-Mouth ใน Community ของแฟนคอนเสิร์ต
- สร้างรายได้ผ่าน Affiliate (โรงแรม บริการเดินทาง และบัตรคอนเสิร์ต)

## Section 2 — Features

| Feature | Description | Input | Output |
|---|---|---|---|
| Concert Trip Budget | คำนวณและสรุปค่าใช้จ่ายของทั้งทริปแบบ Real-time | งบประมาณ, ค่าบัตร, โรงแรม, เดินทาง, Merch, อาหาร | รายการค่าใช้จ่ายทั้งหมด ยอดรวม และสถานะอยู่ในงบหรือเกินงบ |
| Hotel Matching | แนะนำโรงแรมที่เหมาะสมตาม Priority ของผู้ใช้ | Priority (ใกล้ฮอลล์ / คุ้มค่า / รีวิวดี) + จำนวนคืน | รายชื่อโรงแรม ราคา เหตุผลที่แนะนำ และอันดับ |
| Smart Route | วางแผน Timeline เดินทางไป-กลับ | concert (สถานที่ + เวลา) + ประเภทบัตร | Timeline ไป-กลับ + ช่องทางขนส่ง + คำแนะนำขากลับ |
| Prep Checklist | สร้าง Checklist การเตรียมตัวอัตโนมัติ | วันที่จัดงาน ประเภทบัตร ลักษณะงาน (Indoor / Outdoor) | รายการสิ่งของที่ควรเตรียม พร้อมสถานะเช็กได้ ตามสภาพอากาศและสถานที่ |
| Event Policy | สรุปกฎและข้อกำหนดของงาน | ชื่องานคอนเสิร์ต | ของต้องห้าม ขนาดกระเป๋า กล้อง Power Bank และข้อกำหนดอื่นๆ |

## Section 4 — Validate

### User Validation Method

**Target User**
- ผู้หญิง อายุ 20–26 ปี
- มีบัตรคอนเสิร์ตแล้ว
- กำลังวางแผนไปคอนเสิร์ต
- จำนวนผู้ทดสอบ 5 คน

**Validation Method**

1. Task-based Usability Test
ให้ผู้ใช้วางแผนทริปคอนเสิร์ตของตนเอง ตั้งแต่กำหนดงบประมาณ เลือกโรงแรม วางแผนการเดินทาง ไปจนถึงตรวจสอบ Checklist และ Event Policy

2. Think-aloud Protocol
ให้ผู้ใช้อธิบายสิ่งที่กำลังคิดระหว่างใช้งาน เพื่อระบุจุดที่สับสนและ Pain Point ที่พบ

3. Post-test Questionnaire
ประเมินความคิดเห็นของผู้ใช้หลังใช้งานในด้าน
- ความง่ายในการใช้งาน
- ความครบถ้วนของข้อมูล
- ความมั่นใจในการวางแผนทริป
- ความตั้งใจกลับมาใช้งานอีกครั้ง

**Success Metrics**
- Task Success Rate ≥ 90%
- Average Task Completion Time
- Satisfaction Score (1–5)
- Reuse Intention
